PROJECT TITLE:  Instant Motor Claim Estimator
TAGLINE:        AI-powered car damage detection, cost estimation & fraud prevention


--------------------------------------------------------------------------------
PROBLEM STATEMENT
--------------------------------------------------------------------------------

Motor insurance claims are slow, manual, and expensive. Traditional processing
takes 3–15 days with manual assessors, high labor costs, and significant fraud
risk. Insurers lose billions annually from inflated and duplicate claims.

Key problems:
  1. SLOW        — Customers wait 3–15 days for settlement
  2. EXPENSIVE   — Every manual inspection costs Rs.2,000–Rs.8,000
  3. FRAUD-PRONE — Rs.45,000 crore lost annually to fraudulent claims in India


--------------------------------------------------------------------------------
SOLUTION
--------------------------------------------------------------------------------

An AI-powered Instant Motor Claim Estimator that:
  - Analyzes car damage photos in seconds
  - Identifies damaged parts automatically
  - Estimates repair costs intelligently
  - Detects fraud signals proactively
  - Generates an auto-approved or escalated claim decision
  - All running locally — no cloud, no API, no internet required


--------------------------------------------------------------------------------
KEY FEATURES (10 Core Features)
--------------------------------------------------------------------------------

 #   Feature                       Description
 --- ----------------------------- -----------------------------------------------
  1  AI Damage Detection           YOLOv8-based, detects 7 damage types:
                                   bumper, headlight, door, windshield, hood,
                                   scratch, dent
  2  Severity Classification       Minor / Moderate / Severe with 1–10 score
  3  Fraud Detection               Duplicate image hashing, blur scoring, EXIF
                                   metadata analysis
  4  Repair vs Replace Engine      area < 30% → Repair | area >= 30% → Replace
  5  Multi-Image Aggregation       Multiple angles merged, duplicates removed,
                                   confidence averaged
  6  Car Type Pricing              Hatchback / Sedan / SUV / Luxury multipliers
                                   + inflation factor
  7  Confidence-Based Override     AI confidence < 60% → auto-escalate to manual
  8  NLP Explanation Engine        Human-readable paragraph per detection + decision
  9  Claim History + Analytics     SQLite persistence, second dashboard page with
                                   approval rates, cost trends, part frequency
 10  PDF Report Generation         Downloadable report with annotated images,
                                   cost breakdown, and claim decision


--------------------------------------------------------------------------------
TECH STACK
--------------------------------------------------------------------------------

  Layer              Technology
  -----------------  ----------------------------------------
  Frontend UI        Streamlit (Python)
  Computer Vision    YOLOv8 (Ultralytics) + OpenCV
  Cost/Decision      Custom Python rule engine
  Database           SQLite (local, zero-config)
  PDF Generation     ReportLab
  Fraud Detection    MD5 hashing, Laplacian blur, Pillow EXIF
  NLP Explanation    Rule-based template engine (fully offline)


--------------------------------------------------------------------------------
6-STEP PROCESSING PIPELINE
--------------------------------------------------------------------------------

  [1] Upload Photo(s)
         |
  [2] Fraud Check          --> duplicate hash, blur score, EXIF metadata
         |
  [3] AI Detection         --> damaged part + severity + confidence
         |
  [4] Cost Engine          --> base cost x severity x car type x inflation
         |
  [5] Claim Decision       --> Auto Approved / Supervisor / Manual
         |
  [6] PDF Report           --> downloadable, claim saved to SQLite analytics


--------------------------------------------------------------------------------
CLAIM DECISION LOGIC
--------------------------------------------------------------------------------

  Total Estimate         Decision
  ---------------------  ---------------------------
  < Rs.25,000            Auto Approved
  Rs.25,000 - Rs.60,000  Supervisor Review
  > Rs.60,000            Manual Inspection Required
  AI Confidence < 60%    Responsible AI Override --> Manual (regardless of cost)


--------------------------------------------------------------------------------
WHAT MAKES IT STAND OUT
--------------------------------------------------------------------------------

  1. No cloud, no API, no Docker — runs 100% locally on any laptop
  2. Fraud detection built-in — not an afterthought
  3. Responsible AI — confidence override prevents auto-approving uncertain claims
  4. Two detection modes — Smart Simulation (demo) + HuggingFace YOLOv8 (production)
  5. Real-time analytics dashboard — second Streamlit page
  6. NLP explanation layer — transforms it from a tool into an AI Agent


--------------------------------------------------------------------------------
BUSINESS IMPACT
--------------------------------------------------------------------------------

  Metric                 Current           With This System
  ---------------------  ----------------  ----------------------------
  Claim processing time  3–15 days         < 30 seconds
  Assessor dependency    Required          Eliminated for simple claims
  Fraud detection        Manual audit      Automatic (hash+EXIF+blur)
  Cost accuracy          Subjective        Standardized, reproducible

SLIDE 5 — INNOVATION (15 Marks)
--------------------------------------------------------------------------------

WHAT TO SAY:

"What makes our solution different from a simple detection app?"

5 UNIQUE INNOVATIONS:

1. RESPONSIBLE AI DESIGN
   If the model's confidence drops below 60%, the system automatically
   refuses to auto-approve and escalates to human review.
   This is responsible AI — not just AI.

2. FRAUD DETECTION LAYER
   We check for duplicate image submissions, intentional blurring, and
   EXIF metadata to detect edited or reused photos.
   No other student project does this.

3. NLP EXPLANATION ENGINE
   The system explains WHY in natural language:
   "The front bumper covers 42% of surface area, indicating severe damage.
    Replacement is recommended."
   This makes it an AI Agent, not just a model.

4. MULTI-IMAGE AGGREGATION
   Upload 3 angles of the same car. The system deduplicates detections,
   averages confidence, and gives one merged damage report.

5. ANALYTICS DASHBOARD
   Claim history, approval rates, most damaged parts — from local SQLite.
   Shows product thinking beyond just the individual feature.
